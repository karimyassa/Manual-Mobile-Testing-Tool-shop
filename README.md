**Overview**

Manual testing project performed on iPhone 16 (Safari) for PracticeSoftwareTesting.com

The goal was to validate all main user flows, UI behavior, and form validation under a real mobile environment.

**Scope**

Modules Tested:

1.Registration

2.Login

3.Product Listing & Filters

4.Cart

5.Checkout & Payment

**Testing Types:**
Functional • Negative • Boundary • UI/UX (Mobile)

**Environment**
Device	OS	Browser
iPhone 16	iOS 18	Safari v17.x

**Deliverables**

1.Test Cases: Valid & invalid scenarios across all modules

2.Defect Report: Issues with steps, severity & screenshots

(All documents available in this repo.)

**Key Points**

1.100+ test cases executed on mobile Safari

2.Verified full e-commerce flow from registration to payment

3.Documented field validation, UI alignment, and responsiveness issues
